# ace-py

Python FFI bindings for the [Ontolog](https://ontolog-lang.org) ACE interface.

ACE is the foreign function interface for Ontolog. This library makes it easy to expose Python functions to the Ontolog runtime, and to package them as containers.

---

## Installation

```sh
pip install ace-py
```

Or, to install a local checkout for development:

```sh
pip install -e /path/to/ace_py
```

---

## Writing calculations

Decorate any Python function with `@calculation` to register it as callable by the Ontolog runtime. Type annotations are required for all parameters — they tell ace-py how to decode the incoming wire data.

```python
# mycalculations.py
from ace_py import calculation

@calculation
def add(a: int, b: int) -> int:
    return a + b

@calculation
def greet(name: str) -> str:
    return f"Hello, {name}!"

@calculation
def first_n(n: int) -> list:
    return list(range(n))
```

Return values are always treated as a list. If you return a single value it is treated as a one-element list. If you return a `list`, each element becomes a separate output blob.

### Supported types

| Python type | Wire encoding |
|-------------|--------------|
| `str` | UTF-8 bytes |
| `int` | Big-endian two's-complement signed bytes (minimum width) |

## Testing locally

Use `ace-py call` to invoke a calculation from the command line without dealing with the wire format:

```sh
ace-py call mycalculations add 3 5
# 8

ace-py call mycalculations greet World
# Hello, World!

ace-py call mycalculations first_n 4
# 0
# 1
# 2
# 3
```

Arguments are automatically encoded to the correct wire type based on the function's annotations. Results are printed one per line.

## Wire format

This section is relevant if you are implementing an ACE host (e.g. the Ontolog runtime itself) or writing low-level tests.

All integers are **big-endian**. Every message on both stdin and stdout is a length-prefixed blob:

```
[ uint32 length (4 bytes) ][ <length> bytes payload ]
```

**stdin** — one request per invocation:

| Blob index | Content |
|-----------|---------|
| 0 | Function name, UTF-8 encoded |
| 1…N | Arguments, one blob each, encoded per type |

**stdout** — one blob per output item (results are always a list).

**stderr** — human-readable error messages; non-zero exit on failure.

To send a raw request manually:

```sh
ace-py run mycalculations < request.bin > response.bin
```

## Containerizing

`ace-py build` writes a `Containerfile` next to your module and streams the entire build context as a `.tar` to stdout, ready to pipe into any OCI-compatible build tool.

```sh
# Build and load into podman
ace-py build mycalculations | podman build -t my-image -

# Build and load into docker
ace-py build mycalculations | docker build -t my-image -
```

The container accepts one ACE request per invocation via stdin and writes results to stdout:

```sh
echo -n ... | podman run --rm -i my-image > response.bin
```

The generated `Containerfile` can be checked in and customised. To add Python dependencies, commit a `requirements.txt` alongside your module — the build picks it up automatically.
